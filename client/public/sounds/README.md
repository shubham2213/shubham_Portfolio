# Sound Files for Ambient Sound System

This directory should contain three sound files for the portfolio's ambient sound system.

## Required Files

### 1. ambient-hum.mp3
**Purpose:** Continuous background ambient sound  
**Properties:** Loops continuously, volume: 0.15  
**Search on freesound.org:**
- Search terms: "ambient hum sci-fi loop", "low frequency hum loop", "electronic ambient drone", "space ambient loop"
- Filters: CC0 license, loopable
- Recommended duration: 10-30 seconds (will loop seamlessly)
- Look for: Low, subtle, non-intrusive background hum with sci-fi/tech aesthetic

### 2. ui-blip.mp3
**Purpose:** Short sound for hover events  
**Properties:** Single shot, volume: 0.3  
**Search on freesound.org:**
- Search terms: "digital blip UI", "interface beep short", "sci-fi UI sound", "electronic blip", "menu hover sound"
- Filters: CC0 license
- Recommended duration: 0.1-0.3 seconds
- Look for: Short, clean, digital beep/blip sound

### 3. ui-click.mp3
**Purpose:** Short sound for button press/click events  
**Properties:** Single shot, volume: 0.4  
**Search on freesound.org:**
- Search terms: "soft click interface", "UI button click", "digital click sound", "interface confirm sound", "menu click"
- Filters: CC0 license
- Recommended duration: 0.1-0.2 seconds
- Look for: Slightly more pronounced than the blip, satisfying click sound

## How to Find CC0 Sounds on freesound.org

1. Go to https://freesound.org
2. Use the search terms provided above
3. In the filters sidebar, select:
   - License: "Creative Commons 0" (CC0)
   - This ensures the sounds are free to use commercially without attribution
4. Preview sounds and download the ones that fit the aesthetic
5. Rename the downloaded files to match the required filenames above
6. Place them in this directory

## Tips

- **Ambient hum:** Should be very subtle, not distracting. Think "computer room ambience" or "spaceship engine room"
- **UI blip:** Should be pleasant and not annoying, as it plays on every hover
- **UI click:** Should feel satisfying and provide good feedback, but not too loud
- All sounds should fit the futuristic/tech aesthetic of the portfolio

## Testing

After adding the files, test the sound system by:
1. Starting the dev server
2. Clicking the "[ ♪ SOUND ]" button in the navbar
3. The dot should appear: "[ ♪ SOUND • ]"
4. You should hear the ambient hum start playing
5. Hover over interactive elements to test the blip sound
6. Click buttons to test the click sound
